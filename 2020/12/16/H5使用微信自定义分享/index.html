H5使用微信自定义分享

发表于 2020-12-16 分类于 工具类 阅读次数：

主要是对H5页面内嵌到微信端，在微信端使用分享功能时，可以自定义分享标题与内容。

前期配置
  • 配置安全域名
    • 先登录微信公众平台进入“公众号设置”的“功能设置”里填写“JS接口安全域名”。
    • 备注：登录后可在“开发者中心”查看对应的接口权限
  • 引入微信的JSSDK文件：http://res.wx.qq.com/open/js/jweixin-1.2.0.js
    • 在需要调用JS接口的页面引入如下JS文件，（支持https）：http://res.wx.qq.com/open/js/jweixin-1.0.0.js
    • 如需使用摇一摇周边功能，请引入 http://res.wx.qq.com/open/js/jweixin-1.1.0.js
    • 备注：支持使用 AMD/CMD 标准模块加载方法加载
在页面使用

  • 通过config接口注入权限验证配置

    • 所有需要使用JS-SDK的页面必须先注入配置信息，否则将无法调用

    • 同一个url仅需调用一次，对变化url的可在每次url变化之前调用

    • wx.config({

    debug: true, // 开启调试模式,调用的所有api的返回值会在客户端alert出来，若要查看传入的参数，可以在pc端打开，参数信息会通过log打出，仅在pc端时才会打印。

    appId: '', // 必填，公众号的唯一标识

    timestamp: , // 必填，生成签名的时间戳

    nonceStr: '', // 必填，生成签名的随机串

    signature: '',// 必填，签名，见附录1

    jsApiList: [] // 必填，需要使用的JS接口列表，所有JS接口列表见附录2  这里也就是你想要改变哪些位置的自定义分享功能

});

var shareData = {
    title  : '充值送N重豪礼',  //分享标题
    desc   : '充值送N重豪礼',  //分享描述
    link   : rechargegifts.getShareLink(),  //分享链接 该链接域名或路径必须与当前页面对应的公众号JS安全域名一致
    imgUrl : 'https://www.189.cn/client/wap/wapclient/equity/rechargegifts/images/index_banner.png',  //分享图标
};
//分享到朋友圈
wx.updateTimelineShareData(shareData);
//分享给朋友
wx.updateAppMessageShareData(shareData);

wx.error(function(res){
});

wx.checkJsApi({

    jsApiList: ['chooseImage'], // 需要检测的JS接口列表，所有JS接口列表见附录2,

    success: function(res) {

        // 以键值对的形式返回，可用的api值true，不可用为false

        // 如：{"checkResult":{"chooseImage":true},"errMsg":"checkJsApi:ok"}

    }

});

wx.onMenuShareTimeline({  

    title: '', // 分享标题

    link: '', // 分享链接

    imgUrl: '', // 分享图标

    success: function () { 

        // 用户确认分享后执行的回调函数

    },

    cancel: function () { 

        // 用户取消分享后执行的回调函数

    }

});

      分享给朋友:onMenuShareAppMessage

      分享到QQ:onMenuShareQQ

      分享到腾讯微博:onMenuShareWeibo

      分享到Q空间:onMenuShareQZone

后台生成

在前台config配置中的很多内容都是由后台生成的，生成顺序

  • 首先需要Access Token

    由appid和appSecret生成

    获取方式：https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421140183

    调用这个接口就可以获得

  • 需要jsapi_ticket

    使用get方式请求：https://api.weixin.qq.com/cgi-bin/ticket/getticket?access_token=ACCESS_TOKEN&type=jsapi

  • 生成signature签名:

    首先需要 timestamp时间戳，noncestr随机字符串，ticket，url进行调用接口：

    https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=jsapisign

    就可以生成签名

    • 在这里随机字符串，可以通过java代码：UUID.randomUUID().toString

    • 生成时间戳，可以通过java代码：long.string(System.currentTimeMills()/1000)

      时间戳获得的是与格林威治时间相差的毫秒数

    • ticket是我们在第二步生成的，url是我们在调用后台的接口时，传进来的。

最后后台会将生成的这些信息返回，用来前台调用wx.config方法来进行验证权限

可参考链接：http://wapapp03.zsc.189.cn/equity/rechargegifts/rechargegifts_index.html
