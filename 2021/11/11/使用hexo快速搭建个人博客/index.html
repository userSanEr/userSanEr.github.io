2、基本搭建

hexo官网地址：https://hexo.io/zh-cn/

(1)、构建第一步：创建基本项目并启动
  • npm install hexo-cli -g 全局安装
  • hexo init blog 初始化博客
  • cd blog 进入该文件夹
  • npm install 安装相关依赖
  • hexo server 启动项目
(2)、项目目录注释

| - scaffolds md模板

| - source 文章和页面md

| - themes 主题样式

| - _config.yml 这个主体文件夹和公共文件夹下分别有一个都是hexo的核心配置

4、github pages

(1)、建立站点

  • 方式一：

    https://[username].github.io

    仓库名必须为[username].github.io 打包存在master分支

    更适合博客

  • 方式二：

    https://[username].github.io/[仓库名称]

    可以自定义仓库名，打包产物gh-pages分支

    更适合开源项目

(2)、创建仓库

在这里我们使用第一种方式来进行博客搭建

我相信仓库的创建方式大家基本都懂，就不做过多的介绍。

  • 首先创建一个仓库 命名为[username].github.io
  • 接着可以填写一个描述:填写你自己可以看懂的就ok
  • 然后创建仓库
  • 在项目中初始化git仓库 git init
  • 然后添加一个远程仓库 git remote add origin
  • 再我们push到远程仓库之前先要下载一个叫hexo-deployer-git的这个库，这个工具可以帮助我们将生成的代码部署到一个具体的分支

5、自动化部署

(1)、首先我们需要把代码提交到github

因为master分支已经被占用，所以我们需要创建一个新的分支 git checkout -b dev

然后将代码提交到远程仓库

git add .

git commit -m ‘注释语’

git push

接着我们就可以在github上看到提交的代码

(2)、将博客地址配置在仓库首页

可以在code选项这一栏 旁边的about有一个设置的齿轮那里可以进行配置

(3)、使用git actions实现自动化部署

git actions的功能：帮你构建测试部署你的代码、codereview、分支管理、issue的追踪

需要在项目中创建yml文件、在这个文件中配置你需要的工作流

  • 这个文件必须放在.github的workflows文件夹中
  • 然后修改博客内容，之后将你代码提交到dev分支，之后他就会自己完成部署
  • 然后分支页面就会出现一个黄色的小圆点，这就是action起的作用，会自动进行一系列的工作流，自动部署的博客
  • 点击小圆点的detail一系列东西就可以看到他进行的内容
  • 之后我们就可以查看博客看一下是不是真的成功了

6、在线编辑文章

可以点击github自带的编辑功能，在修改之后进行保存，然后commit，github就会自己进行一系列的工作流了。
